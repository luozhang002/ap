# 客户走访管理系统 - 产品需求文档

## 产品概述

客户走访管理系统是一个帮助销售人员高效管理客户走访的Web应用。系统支持Excel批量导入客户信息，通过地图可视化展示周边客户分布，提供距离筛选和智能路线规划功能，帮助销售人员优化走访路径，提高工作效率。

## 核心功能

### 1. 客户信息管理
- **Excel导入**：支持批量导入客户信息（企业名称、企业地址、授信额度、联系电话）
- **客户列表**：查看所有客户信息，支持搜索和筛选
- **客户详情**：查看客户详细信息，包括走访记录

### 2. 地图可视化
- **当前位置定位**：自动获取并显示用户当前位置
- **客户位置标记**：在地图上标记客户位置，区分已走访/未走访状态
- **距离筛选**：支持按距离筛选客户（1公里内、2公里内、5公里内、不限距离）

### 3. 走访管理
- **走访计划**：选择多个客户创建走访计划
- **路线规划**：自动生成最优走访路线
- **状态标记**：标记客户走访状态（已走访/未走访）
- **走访记录**：记录每次走访的详细信息

### 4. 导航功能
- **跳转导航APP**：一键跳转到第三方导航应用
- **途经点设置**：将选中的客户设置为途经点
- **最优路线**：生成包含所有途经点的最优行进路线

## 用户故事

### 用户角色：销售人员

1. **导入客户信息**
   - 作为销售人员，我希望能够通过Excel批量导入客户信息，以便快速建立客户数据库

2. **查看周边客户**
   - 作为销售人员，我希望在地图上看到我周边的客户分布，以便规划走访路线

3. **筛选距离**
   - 作为销售人员，我希望能够按距离筛选客户，以便优先走访附近的客户

4. **规划走访路线**
   - 作为销售人员，我希望系统能帮我生成最优的走访路线，以便节省时间和精力

5. **跳转导航**
   - 作为销售人员，我希望能够一键跳转到导航APP并自动设置途经点，以便开始走访

6. **标记走访状态**
   - 作为销售人员，我希望能够标记客户的走访状态，以便跟踪工作进度

## 页面结构

### 1. 首页 (`/`)
- 系统概览
- 快速操作入口
- 今日待走访客户统计
- 最近走访记录

### 2. 客户列表页 (`/customers`)
- 客户信息列表
- 搜索和筛选功能
- Excel导入按钮
- 客户状态标识

### 3. 地图页面 (`/map`)
- 全屏地图展示
- 当前位置标记
- 客户位置标记（已走访/未走访）
- 距离筛选器
- 走访计划创建
- 导航跳转按钮

### 4. 客户详情页 (`/customers/:id`)
- 客户基本信息
- 走访历史记录
- 走访状态标记
- 地图位置查看

## 数据模型

### Customer（客户）
```typescript
{
  id: string;
  companyName: string;        // 企业名称
  address: string;            // 企业地址
  creditLimit: number;        // 授信额度
  contactPhone: string;       // 联系电话
  latitude: number;           // 纬度
  longitude: number;          // 经度
  visitedStatus: 'visited' | 'unvisited';  // 走访状态
  lastVisitedAt: Date;        // 最后走访时间
  createdAt: Date;
  updatedAt: Date;
}
```

### VisitRecord（走访记录）
```typescript
{
  id: string;
  customerId: string;         // 客户ID
  visitedAt: Date;            // 走访时间
  notes: string;              // 走访备注
  createdAt: Date;
}
```

## API 端点

### 客户相关
- `POST /api/customers/list` - 获取客户列表
- `POST /api/customers/create` - 创建客户
- `POST /api/customers/get` - 获取客户详情
- `POST /api/customers/update` - 更新客户信息
- `POST /api/customers/import` - Excel导入客户信息
- `POST /api/customers/nearby` - 获取周边客户（按距离筛选）

### 走访记录相关
- `POST /api/visits/list` - 获取走访记录列表
- `POST /api/visits/create` - 创建走访记录
- `POST /api/visits/get` - 获取走访记录详情

## 技术要点

### 地图服务
- 使用腾讯地图/高德地图 JavaScript API
- 地理编码：将地址转换为经纬度坐标
- 逆地理编码：将坐标转换为地址
- 路线规划：生成最优驾车路线
- 导航跳转：构造导航URL，支持途经点

### Excel导入
- 前端使用 SheetJS (xlsx) 解析Excel文件
- 后端批量创建客户记录
- 地理编码批量处理地址

### 距离计算
- 使用 Haversine 公式计算两点间距离
- 支持按距离排序和筛选

### 导航跳转
- 支持主流导航APP（高德地图、百度地图、腾讯地图）
- 构造带途经点的导航URL
- 适配移动端和桌面端

## 设计风格

- **主色调**：专业蓝色系，体现商务稳重
- **辅助色**：绿色（已走访）、橙色（未走访）
- **视觉层次**：清晰的卡片式布局，突出地图展示
- **交互设计**：流畅的动画效果，直观的操作流程
